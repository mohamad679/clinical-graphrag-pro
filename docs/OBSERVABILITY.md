# Observability and Performance Monitoring

Clinical GraphRAG Pro includes advanced monitoring, tracing, and metric collection capabilities to measure performance, compute token costs, and trace agent execution steps.

This is a portfolio and synthetic-data demonstration. Observability features are for engineering inspection and regression testing; they are not a compliance logging program by themselves.

## Observability Modes

Set `OBSERVABILITY_MODE` to control how much raw text may appear in logs and traces:

| Mode | Intended environment | Raw text handling |
| --- | --- | --- |
| `LOCAL_SYNTHETIC_DEBUG` | Local synthetic development | Keeps useful synthetic debugging detail while redacting known secrets and tokens. |
| `STAGING_REDACTED` | Staging-like demos | Redacts secrets and sensitive path-like fields. |
| `PRODUCTION_METADATA_ONLY` | Production-like review | Redacts raw patient queries, document chunks, prompts, tool output, full answer text, and sensitive paths; stores metadata such as request ID, tenant-safe IDs, stage, latency, model identifier, retrieval counts, validation status, abstention status, and error category. |

Do not use local debug logs as evidence of HIPAA/GDPR compliance, and do not process real patient data without independent controls outside this repository.

## 1. Metrics Endpoint

FastAPI exposes an operational metrics summary endpoint. This endpoint yields rolling-window statistics for latency, token counts, and cost.

### Querying the Metrics Endpoint

**Request:**
```bash
curl -X GET "http://localhost:8000/api/v1/admin/metrics" \
     -H "Authorization: Bearer <YOUR_JWT_TOKEN>"
```

**Expected JSON Response Format:**
```json
{
  "dense_search_latency_ms_avg": 15.2,
  "sparse_search_latency_ms_avg": 8.4,
  "rerank_latency_ms_avg": 22.1,
```

## 2. Telemetry and Latency Breakdown

Metrics are recorded across the core service layers:
1. **Dense Search (Vector Store)**: Tracks vector search latency and the count of matching chunks returned.
2. **Sparse Search (BM25)**: Monitors BM25 index query performance.
3. **Reranker (Cross-Encoder)**: Profiles reranking times for retrieved clinical contexts.
4. **Graph Queries (Neo4j / SQL)**: Latency profiling on clinical graph and temporal relationships lookup.
5. **LLM Calls**: Evaluates model generation times, records provider connection errors, and tracks token consumption.
6. **RAG Pipeline**: Tracks total source citations and counts queries leading to no-context or abstention decisions.

## 3. Real-Time Tracing with SSE (Server-Sent Events)

When run, the agent orchestrator publishes rich execution trace events to the SSE channel `/api/v1/agents/run`. Reviewers can inspect the following events:

- `plan_created`: The initial workflow steps generated by the supervisor.
- `tool_selected`: The agent routing to a specific retrieval or clinical tool.
- `tool_start` & `tool_complete`: Tracks start/stop and records the execution duration for each tool.
- `evidence_collected`: Emitted when new context snippets (chunks, relations) are fetched.
- `answer_drafted`: Emitted when the first answer candidate is synthesized.
- `verification_passed` / `verification_failed`: The result of the red-team evaluator evaluation, including reason flags.
- `retry_triggered`: Emitted if the critic rejects a draft but there are remaining attempts left.
- `abstention`: The agent gracefully failing closed (e.g. `INSUFFICIENT_EVIDENCE` or safety gates).

These events are rendered dynamically on the **Trace Stream Canvas** in the frontend client.
